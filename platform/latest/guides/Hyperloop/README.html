Introduction

The Hyperloop Module gives you direct access to the native APIs. Titanium SDK already covers plenty of the native APIs and makes them cross-platform addressing the most popular APIs. This module was built for those developers who require access to even the most obscure APIs out there.

Hyperloop is available to users who have Pro and Enterprise subscriptions. Hyperloop is also available for Trial users.

Installation

Prior to SDK 5.4.0, you will need to follow these steps to install the beta version of Hyperloop:

  1. Download the Hyperloop module and plugin from Appcelerator Labs Hyperloop Module.

  2. Place the downloaded files in the correct directories In your user directory.

    1. On macOS, that directory should be: /Users/<user>/Library/Application Support/Titanium

  3. Install cocoapods version 0.39.0 (the latest version 1.0.1 is currently not supported):

    1. Install with brew by going to the terminal and executing brew search cocoapods. If cocoa pods is not shown, use brew update.

    2. Next, run this command: brew install Caskroom/versions/cocoapods0

    3. Finally, check to confirm that you have version 0.39.0: pod —version

      1. If the versions says 1.0.1 then use brew switch cocoapods 0.39.0 to switch it to a compatible version.

Hyperloop is available to all Pro and Enterprise users of Titanium SDK 5.4.0 and beyond.

Requirements

You’ll need to have the following minimum requirements to use Hyperloop for iOS:

  • Titanium 5.4.0+

  • Alloy 1.8.0+

Hyperloop only works with Titanium applications that are registered with the platform.

Sample projects

See Hyperloop Sample Apps page for details on how to build a Hyperloop app.

Windows developers should refer to Windows Runtime Direct API Access the Windows equivalent for Hyperloop.

Known issues

CocoaPods version 1.0.x is not supported by Hyperloop (JIRA Issues Macro: The JIRA server returned a trusted apps error: USER_UNKNOWN; Unknown User: {0}; ["doc-robot"]).

  • CocoaPods 0.39 (< 1.0.0): A deprecated warning is thrown during build process but should still work

  • CocoaPods 1.0.1 (>= 1.0.0): Works

  • For both: The charts example (implemented using a Cocoapods-library) works
