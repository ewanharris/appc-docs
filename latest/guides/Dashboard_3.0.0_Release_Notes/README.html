Dashboard 3.0.0 Release Notes

The 3.0.0 release of Appcelerator Dashboard contains several bug fixes and a few new features and improvements.

  • New Features and Improvements

  • Other Changes

  • Bug Fixes

  • Known Issues

New Features and Improvements

  • Added ability to filter ACS data object logs tables by both time and date that the object was created or last updated. Previously, you could only filter by date.

  • Improved usability of form used to manage application team members.

Other Changes

  • If an app consumes an API defined in another Dashboard organization, that API is listed but not linked to.

  • For VPC environments, the application Environment menu now displays a label instead of a dropdown menu. For non-VPC environments, the dropdown menu continues to display 'Production' and 'Development' menu items.

  • Added a 'Cancel' button to the 'Add Member' form available when creating a new organization.

Bug Fixes

Summary

Fixed issue with Manage Data displaying data for apps that consume multiple APIs.

Fixed issue with spacing of menu items in API dropdown in Firefox.

Fixed an issue displaying incorrect data when filtering event funnels data on different date ranges.

Fixed issue with custom Dashboard app icons being replaced by generic Alloy/Titanium icons.

The "Details" link to the Cloud service in Appcelerator Studio now opens the correct page.

ACS and Node.ACS documentation not visible to non-staff Dashboard users

Restored selection highlight for secondary navigation.

Fixed typo in instructions for enabling test services for native Android applications.

Fixed issue with Insights where Insights-only users can login to Insights but cannot select apps to analyze.

The 'Quick Start Guide' link for the ACS documentation now opens the expected page.

The refresh icon on the Node.ACS Logs table has been restored.

The sorting of application members on an application's Team tab has been fixed.

The 'Docs' link in the Dashboard's bottom navigation bar now opens the Dashboard documentation page, as expected.

Known Issues

Summary

In Node.ACS logs, long URLs aren't getting properly wrapped/shortened.

Column widths change unexpectedly while scrolling some tables.

On Manage Orgs, using the up/down arrow keys to navigate the menu of organizations does not produce expected results.

The 'Date Created' field for newly created APIs is not visible.
